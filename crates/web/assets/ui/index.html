Welcome to Indexa — your local context engine

Point Indexa at a folder and it builds a searchable context map of everything inside — entirely on your machine. After that, your AI tools answer from it instead of re-reading your files every time.

Understands code, presentations, Word documents, PDFs, notes, images, and more — indexed as first-class content.

  1. Add a folder. A code repo, your Documents, a notes vault — anything.
  2. Indexa builds context. It scans, embeds, and summarizes locally with Ollama. No files leave your machine.
  3. Ask or export. Query in plain language, or export a context file to Claude, Cursor, or any AI tool.
or run indexa scan <path> in your terminal

Context map

This map shows how ready your files are for AI to use. Each box is a folder, sized by how much content it holds, and colored by readiness:

  • ● Built — summarized and searchable. Ask about it and you'll get good answers.
  • ◐ In progress — being indexed right now; refreshes to green when done.
  • ○ Not built — not indexed yet. Hover the folder in the sidebar and click Index for search to build it.
  • ✕ Failed — something went wrong; click the ✕ in the sidebar to retry.

The words: a chunk is a small searchable piece a file is split into (so AI can find the exact relevant bit, not re-read the whole file); a summary is a short AI-written description of a file or folder; coverage is the share of your folders that have summaries built. The Graph tab shows which files call into which.

Built In progress Failed Not built · sized by chunks indexed

This is a map of how your files connect. Each dot is a file, and a line between two files means one of them uses code from the other.

  • Bigger dots are more important — lots of other files rely on them, so they're the hubs of your project. Small dots are used by fewer files.
  • Solid lines are confident connections (the files clearly reference each other, or live together). Dotted lines are best-guess connections matched only by a shared name — treat them as approximate.

Try it: click a dot to focus on it and dim everything else, then Expand neighbors to see just its direct connections. Use Scope above to zoom into one folder, and Show all in scope to go back.

Nothing here changes your files or your answers — it's a read-only view of what's already indexed.

Call edges link a file to any in-scope file that defines a function it calls — matched by bare name (case-sensitive). Languages: Rust, Python, JS, TS, Go, Java.

Advanced Features

ANN search

Speeds up dense retrieval on indexes with ≥ chunks. Rebuilt automatically when the chunk count changes. Restart required.

Image captions
Vision model

Runs a vision model alongside the summary models during indexing. The memory watchdog now counts it and warns here if the combination won't fit your budget. Applies on the next indexa deep run.

Audio transcription
whisper binary

Requires whisper-cli (whisper.cpp) installed separately. Applies on the next indexa deep run.

Video captions
Vision model

Requires ffmpeg (frame extraction) + an Ollama vision model. Samples 1 frame/2s, max 8 frames per video — frames are captioned one at a time, so the memory peak is a single frame's model (not 8×). The memory watchdog now counts that vision model and warns here if it won't fit.
